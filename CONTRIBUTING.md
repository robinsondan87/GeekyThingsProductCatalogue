# Contributing

## Commit messages
Use clear, structured commit messages:

Format:
- `Type: short summary`

Types:
- `Major` for breaking changes or big restructures
- `Minor` for new features
- `Fix` for bug fixes
- `Docs` for documentation updates
- `Chore` for tooling or maintenance

Examples:
- `Major: restructure product folders and regenerate CSV`
- `Minor: add product detail page and README editor`
- `Fix: correct SKU renaming logic`
- `Docs: add changelog and contribution guidance`
